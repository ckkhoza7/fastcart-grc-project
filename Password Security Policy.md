**Password Security Policy**

**Purpose:** The purpose of this Password Security Policy is to clearly define the requirements for creating, managing, and protecting user passwords in order to reduce access to FastCart.Inc user information, data, and the overall system.

**Scope:** This policy applies to all employees, contractors, and third-party users who access FastCart.Inc systems, including internal applications, cloud services, and administrative accounts

**Policy Statements:**

- Password Requirements  
  - Passwords must be at least 12 characters in length  
  - Password reuse across multiple platforms is prohibited  
  - Repeating numbers (ex.123) are prohibited in the use of passwords  
  - Passwords must include uppercase, lowercase, numbers, and special characters  
- Password Management  
  - Passwords must not be shared between users  
  - Default passwords must be changed upon first login  
- Account Security Controls  
  - Multi-factor Authentication must be applied to all administrative accounts  
  - Accounts will be locked after multiple failed attempts  
  - Passwords must be stored using secure hashing  
- Prohibited Behavior  
  - Writing down passwords in an unsecured location is prohibited  
  - Sharing credentials via email or other messaging platforms is prohibited

**Enforcement:** Violations of this policy might include disciplinary actions, including the loss of access or in some cases termination of employment  
**Review Cycle:** This password policy will be reviewed annually or updated based on new security requirements
