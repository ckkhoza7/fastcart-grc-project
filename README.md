FastCart Inc – GRC Cybersecurity Risk Assessment Project

Overview
FastCart Inc is a simulated mid-size e-commerce company designed to demonstrate Governance, Risk, and Compliance (GRC) cybersecurity principles in a real-world environment.

This project models a cloud-based retail organization that processes customer transactions, stores sensitive data, and operates web-based services.

The purpose of this project is to identify cybersecurity risks, evaluate their impact, and map them to industry-standard security frameworks.

Company Profile
FastCart Inc is an e-commerce company that sells consumer goods such as clothing, electronics, and household items through an online platform.

Company Details
- Size: 80 employees (8 IT staff)
- Industry: E-commerce / Retail
- Infrastructure: Cloud-based

Technology Stack
- Cloud Platform: Amazon Web Services (AWS)
- Operating Systems: Linux servers, Windows endpoints
- Database: MySQL
- Monitoring & Logging: Splunk

 Key Assets
- Customer database (PII and payment information)
- Web application platform
- Administrative accounts
- Employee endpoints
- Password authentication systems

Security Posture

FastCart Inc currently operates with a basic security maturity level and limited enterprise security controls.

Key assumptions include:
- No Multi-Factor Authentication (MFA)
- No centralized Identity and Access Management (IAM)
- No Endpoint Detection and Response (EDR)
- Weak password policies
- Limited security monitoring and log review processes

Risk Assessment Summary

A formal risk assessment was conducted to identify threats, vulnerabilities, and potential business impact.

Key threats analyzed include:
- Brute force attacks
- Password spraying and credential stuffing
- Phishing attacks
- Ransomware
- Data exfiltration
- Web application attacks (SQL Injection, XSS)
- Insider threats
- Privilege escalation
- Device theft and malware infection

Each risk was evaluated using:
- Likelihood
- Impact
- Risk rating (Low, Medium, High, Critical)

Framework Alignment

This project is aligned with the **NIST Cybersecurity Framework (CSF)**:

- Identify – Asset and risk identification
- Protect – Security controls (password policy, access control)
- Detect – Monitoring and logging (Splunk)
- Respond – Incident handling procedures
- Recover – System restoration and backups

Included Documents

- Company Overview
- Risk Assessment Matrix
- Password Security Policy
- Incident Response Plan

Key Security Findings

- Lack of MFA significantly increases risk of account compromise
- Weak password practices enable credential-based attacks
- Limited monitoring reduces detection capability for active threats
- Absence of endpoint protection increases malware risk
- Insufficient access control increases risk of insider threats

Purpose of This Project

This project was created to demonstrate practical understanding of:
- Cybersecurity risk analysis
- GRC (Governance, Risk, Compliance) principles
- Security policy development
- Incident response planning
- NIST Cybersecurity Framework mapping

Skills Demonstrated

- Risk identification and analysis
- Threat modeling
- Security policy writing
- Incident response planning
- Framework mapping (NIST CSF)
- Documentation for cybersecurity governance


This is a simulated environment designed for educational and portfolio purposes.
